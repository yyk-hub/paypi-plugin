PayPi Documentation

Complete guide to integrating Pi Network payments into your application.

Quick Start

Get started with PayPi in 5 minutes:

1. Register for Merchant Account

Visit the Merchant Portal and create an account.

2. Get Your API Key

After registration, you'll receive your API key and merchant ID.

3. Add Credits

Deposit Pi to get credits (1π = 1 credit). Each payment costs 2% in credits.

4. Make Your First Payment

POST /api/pi/create
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY

{
  "amount": 10.0,
  "description": "Product purchase",
  "merchant_id": "YOUR_MERCHANT_ID"
}

Payment Flow Overview

PayPi handles the entire Pi payment flow for merchants. The typical payment flow is:

  1. Merchant server creates payment via /api/pi/create
  2. PayPi creates order and Pi payment request
  3. User approves payment in Pi Browser
  4. PayPi approves and completes the payment
  5. Blockchain transaction is submitted
  6. PayPi sends webhook notification to merchant server
  7. Merchant fulfills the order
Note: Merchants only need to call the create payment API. PayPi handles approval, completion, blockchain transactions, refunds, and credits automatically.

Authentication

All API requests require authentication using your API key:

Authorization: Bearer YOUR_API_KEY
Security: Never expose your API key in client-side code. Always make API calls from your server.

Credits System

PayPi uses a prepaid credits system:

  • Deposit: 1π = 1 credit
  • Payment Fee: 2% (0.02 credits per 1π)
  • Refunds: Credits are returned on refund
Example: To process 100π in payments, you need 2 credits (100π × 2% = 2 credits).

Create Payment

Create a new payment request:

Endpoint

POST /api/pi/create

Request Body

{
  "amount": 10.0,
  "description": "Product XYZ",
  "merchant_id": "YOUR_MERCHANT_ID"
}

Response

{
  "success": true,
  "payment_id": "ZFYX1wIsrV5GuNT8y718TwaUcaWY",
  "order_id": "ORD1773837119058",
  "amount": 10.0,
  "approval_url": "https://..."
}

Next Steps

  1. Redirect user to approval_url
  2. User approves in Pi Browser
  3. Payment automatically completes
  4. Webhook notification sent (if configured)

Complete Payment

Payment completion is automatic after user approval. Check status:

Endpoint

GET /api/pi/payment/{payment_id}

Response

{
  "payment_id": "ZFYX1wIsrV5GuNT8y718TwaUcaWY",
  "status": "completed",
  "amount": 10.0,
  "txid": "fcac41d4765e4a6ce808c9e56fcc1b2a...",
  "completed_at": 1773837200
}

Process Refund

Refund a payment (full or partial):

Endpoint

POST /api/refund/process

Request Body

{
  "order_id": "ORD1773837119058",
  "amount": 10.0,
  "reason": "Customer request"
}

Response

{
  "success": true,
  "refund_id": "REF_1773837300_abc",
  "txid": "89265e67e8515aa18ff43543cafc...",
  "amount": 10.0,
  "refund_type": "full",
  "credits_refunded": 0.2
}
Partial Refunds: You can refund any amount up to the remaining balance. Multiple partial refunds are supported.

Integration Guide

Step 1: Server-Side Setup

// Node.js example
const axios = require('axios');

const API_KEY = process.env.PAYPI_API_KEY;
const MERCHANT_ID = process.env.PAYPI_MERCHANT_ID;
const API_URL = 'https://paypi-plugin.pages.dev/api';

async function createPayment(amount, description) {
  const response = await axios.post(`${API_URL}/pi/create`, {
    amount,
    description,
    merchant_id: MERCHANT_ID
  }, {
    headers: {
      'Authorization': `Bearer ${API_KEY}`,
      'Content-Type': 'application/json'
    }
  });
  
  return response.data;
}

Step 2: Client-Side Integration

// After user clicks "Pay with Pi"
async function handlePayment() {
  // Call your backend
  const payment = await fetch('/api/create-payment', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      amount: 10.0,
      description: 'Product purchase'
    })
  }).then(r => r.json());
  
  // Redirect to Pi Browser
  window.location.href = payment.approval_url;
}

Step 3: Handle Callback

// User returns after approval
app.get('/payment/callback', async (req, res) => {
  const { order_id } = req.query;
  
  // Verify payment completed
  const order = await getOrder(order_id);
  
  if (order.status === 'completed') {
    // Deliver product/service
    await fulfillOrder(order_id);
    res.redirect('/success');
  } else {
    res.redirect('/error');
  }
});

Webhooks

PayPi sends webhook notifications when payment status changes.

Payment Completed Webhook

{
  "event": "payment.completed",
  "order_id": "ORD123",
  "payment_id": "ABC123",
  "amount": 10.0,
  "txid": "stellar_tx_hash",
  "status": "completed"
}

Refund Completed Webhook

{
  "event": "refund.completed",
  "refund_id": "REF123",
  "order_id": "ORD123",
  "amount": 5.0,
  "txid": "stellar_tx_hash"
}
